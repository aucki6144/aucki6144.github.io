Spark: Alogrithm Design for Big Data

Embarrassingly parallel problems

Problems that can be easily decomposed into many independent problems. For example:

  • Word Count
  • K-means
  • PageRank

Classical Divide-and-Conquer

Divide problem into 2 parts → Recursively solve each part → Combine the results together.

D & C under big data systems

  • Divide problem into ppp partitions, where (ideally) ppp is the number of executors in the system.
  • Solve the problem on each partition.
  • Combine the results together.

Example: sum(), reduce()

Prefix Sums

Input: Sequence xxx of nnn elements, binary associative operator +

Output: Sequence yyy of nnn elements, with yk=x1+...+xky_k=x_1+ ... + x_kyk​=x1​+...+xk​

Example: x=[1,4,3,5,6,7,0,1]x=[1,4,3,5,6,7,0,1]x=[1,4,3,5,6,7,0,1] y=[1,5,8,13,19,26,26,27]y=[1,5,8,13,19,26,26,27]y=[1,5,8,13,19,26,26,27]

Sample Sort

Step1: Sampling

Collect a sample of 4p ln(p/2)4p\ ln(p/2)4p ln(p/2) elements

Step2: Broadcast splitters

Pick every (i×4ln(p/2))−th(i\times4ln(p/2))-th(i×4ln(p/2))−th element in the sample as spliters. Then broadcast them to all machines.

Step3: Shuffling

Each machine partitions its data using the splitters. Send data to the target machine.

Step4: Sort each partition

Each machine sorts all data received.

Q: How to sample one element uniformly from nnn elements stored on ppp servers?

A:

  • First randomly sample a server (partition)
  • Ask that server to return an element randomly chosen from its n/pn/pn/p elements.
  • The probability of each element being sampled is 1p×pn=1n\frac{1}{p}\times \frac{p}{n}=\frac{1}{n}p1​×np​=n1​

Q: How to sample many elements at once?

A: Do each of the two steps above in batch mode

  • First sample plnpplnpplnp servers with replacement (this can be done at the master node).
  • If a server is sampled kkk times, we ask that server to returan kkk samples (with replacement) from its local data.
